## Flow2Go PRD（产品需求文档）

### 1. 产品概述

**产品定位**：浏览器端大图/流程图编辑器，交互风格接近 Figma，可快速产出“节点 + 边 + 画框(Frame) + 素材”的说明图并保存/分享。  
**核心理念**：Frame 是真实容器（有父子关系与局部坐标），不是“视觉包裹 + 假链接”。

### 2. 目标用户

- **产品/运营/售前**：需要快速搭建流程/业务说明图
- **研发/架构**：需要画模块关系/调用链路/系统结构
- **设计/交付**：需要用素材增强表达，输出可复用模板

### 3. 使用场景

- 新用户打开即看到可编辑示例，立刻上手
- 用 Frame 将复杂图分区：A 包 B 包 C，拖动重排层级像 Figma
- 复杂内容编组：群组内有群组、节点、边、素材，整体打包与再嵌套
- 对边写说明：双击修改 label；边菜单只保留简单必需项
- 导入 SVG/PNG 素材：可旋转、翻转、（SVG）覆盖颜色

### 4. MVP 范围（必须实现）

#### 4.1 画布基础能力

- 节点：创建、拖拽、缩放、框选、多选、复制粘贴、撤销/重做
- 边：连接、选择、删除、（可选）动画
- 视图：缩放/平移、MiniMap、Controls、背景网格/点阵
- 右键菜单：画布/节点/边分别弹出

#### 4.2 Frame（画框）容器能力（必须对齐 Figma）

- Frame 为真实容器，使用 `parentId` 建立父子关系
- 子节点 position 为父容器的**局部坐标**
- Frame 可嵌套：Frame 可以包含 Frame
- 拖入/拖出/重挂载规则详见 `02-Interaction.md`

#### 4.3 Group（群组/编组）

- 多选 ≥2 可编组
- 单选一个 group 允许“再包一层”（用于第三层群组）
- 新群组 bounds 必须正确包含：
  - 选中节点本体
  - 若选中包含群组：递归包含其子孙
  - 相关边的几何（waypoints）并加安全 padding

#### 4.4 边（Edge）编辑（简洁版）

点击边弹出菜单（紧凑 popup），仅保留：
- Label：开关（开=显示，文字编辑通过双击）
- 线类型：smoothstep / bezier
- 箭头类型：无/终点/起点/双向
- 线条颜色（统一 ColorEditor）
- 线宽
- 动画开关

#### 4.5 素材（Asset）

- 导入 SVG/PNG 作为节点
- 支持：
  - 45° 步进旋转
  - 水平翻转 / 垂直翻转
  - SVG 颜色覆盖：使用同款 ColorEditor 弹出拾色板（portal）

#### 4.6 文本编辑互斥（全局锁）

任何文字编辑开始时：
- 只允许出现文字编辑相关 UI
- 必须关闭并禁止其他节点/边/Inspector 菜单复活（包括颜色拾色板交互）

#### 4.7 默认示例

新用户空快照进入编辑器时，加载默认示例（`defaultExample.json`）。

### 5. 非目标（暂不做）

- 多人协作/实时同步
- 账号权限体系（除基础登录页外）
- 自定义插件体系
- 复杂模板市场

### 6. 验收标准（高层）

满足 `04-QA.md` 全部用例，且以下“关键体验”必须一致：
- Frame 拖入/拖出/重挂载像 Figma（以光标落点表达意图）
- 编组不跳位、不产生层级循环，bounds 包含边
- 线上发布后箭头 marker 不丢
- 触摸板缩放/滚动稳定，不被页面滚动抢占
