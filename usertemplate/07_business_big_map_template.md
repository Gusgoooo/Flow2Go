# Business Big Map Template（业务大图）

## 适用场景
适用于战略全景图、能力地图、知识结构图、系统分层大图、方案拆解图、业务总览图。

## 模板目标
生成一张 **大图层级排版** 的结构图：按“章节 frame → 模块 group → 子模块 subgroup → 要点 quad”组织内容，强调板块感、分区感、层级感，适合 Flow2Go 大画布继续编辑。

## 强约束
1. 必须使用多层嵌套的 subgraph 来表达层级（frame/group/subgroup）。
2. 整体以“章节”方式组织：多个一级 subgraph 代表多个章节 frame。
3. 每个章节必须选择一种主要“画框类型”（Type A~F）作为该章节内部排版风格，并保持一致。
4. 优先保留结构语义，禁止为了视觉均匀强行拉平层级。
5. 节点标题必须简短、专业；长语义要拆分为主/副标题（V2）。
6. 任意父画框下，直接子画框数量不得超过 3（硬限制）。
7. 节点布局统一为“竖向倒N”：先上下，再换列；节点列数最多 2 列。
8. 子画框布局优先横向平铺（同层先横向扩展），然后再换行。

## 画框类型字典（给模型的心智）
- Type A：横向概览条（并列概览、阶段摘要）
- Type B：双栏模块板（左右两域对照）
- Type C：多列卡片组（多模块并列，每模块 2~3 要点）
- Type D：左标题章节块（章节式信息区）
- Type E：极简横向承载（摘要/索引/结论条）
- Type F：双层嵌套复杂区（二级以上嵌套的复杂信息密集区）

## 推荐用户模板
```md
请为以下内容生成一张适合 Flow2Go 大图画布展示的层级结构图（业务大图）。

【主题】
{{topic}}

【内容说明】
{{description}}

【任务目标】
根据输入内容的层级关系、并列关系、信息密度与嵌套深度，自动选择最合适的画框类型（Type A~F）来承载内容，并生成一张从上到下排布的大图结构。

【核心规则（强制）】
1. 这是“div 式层级大图”，不是普通流程图。
2. 必须先判断内容结构，再选择 frame type；并在每个一级 subgraph 标题中标注（例如：`subgraph 章节名｜Type F`）。
3. 必须使用 subgraph 嵌套表达层级：frame(章节) → group(模块) → subgroup(子模块) → quad(要点)。
4. 同一章节内尽量使用统一内部栅格与等距对齐；结构优先于装饰。
5. 长文本必须用 V2 主/副标题编码（`主标题｜副标题`）。
6. 每个父画框直接子画框最多 3 个。
7. 节点必须按“竖向倒N”排列（先上下，再换列），且最多 2 列。
8. 子画框必须优先横向平铺，避免单列堆叠。
9. 三种 case 必须混合出现：
   - Case 1：1层嵌套（章节 -> 节点）
   - Case 2：2层嵌套（章节 -> 二级画框 -> 节点）
   - Case 3：3层嵌套（章节 -> 二级画框 -> 子画框 -> 节点）

【增强要求】
- 自动补齐合理的中间层级与缺失模块分组
- 自动避免过密排版
- 少箭头、少连线：更多用分组承载“属于/包含”关系；只保留关键跨模块关系边
- 输出更像战略图/能力地图/结构大图，适合继续编辑与扩图

【嵌套一致性约束（新增，强制）】
- 任意父画框（frame）下，直接子画框必须保持相同的“相对嵌套层级”：
  - 要么该父画框下所有直接子画框都只到“二级画框 -> quad”（即全员 Case 2）
  - 要么该父画框下所有直接子画框都必须再包含子画框 -> quad（即全员 Case 3）
- 禁止同一父画框下同时出现 Case 2 与 Case 3 的混搭。
- 禁止同一父画框下“同时直接包含 quad（Case 1 形态）”和“同时包含子画框（Case 2/3 形态）”。
```
