# Frontend-Backend Flow Template

## 适用场景
适用于：
- 前后端调用关系
- 页面交互与服务处理链路
- BFF / API Gateway / 微服务交互
- 用户操作到数据库落库的完整链路
- Web / App / Admin 的请求处理流程

## 模板目标
将用户输入的业务需求，生成一张 **前端—中间层—后端—数据层** 结构明确的流程图，突出请求流、响应流、状态变化、异常处理与依赖关系。

## 强约束
1. 必须按层分组，至少包含以下 4 个区域：
   - Client / Frontend
   - Interface Layer / BFF / Gateway
   - Backend Services
   - Data / External Dependencies
2. 至少包含：
   - 1 个用户触发节点
   - 2 个前端处理节点
   - 2 个后端服务节点
   - 1 个数据存储节点
   - 1 个异常或失败分支
3. 必须体现：
   - 用户操作
   - 参数组装或表单校验
   - 接口请求
   - 服务处理
   - 数据读写
   - 响应返回
4. 若场景合理，应自动补齐：
   - 鉴权
   - 缓存
   - 日志
   - 重试 / 降级
5. 前后端之间的边必须有语义，例如：
   - 发起请求
   - 返回结果
   - 鉴权校验
   - 查询数据
   - 写入记录

## 输出风格偏好
- 主链路从左到右或从上到下
- 前端与后端分区明显
- 数据层放在最右或最下
- 异常路径放侧边，避免污染主链

## 推荐用户模板
```md
请为以下业务生成一张适合 Flow2Go 展示的 Frontend-Backend Flow 图。

【业务主题】
{{topic}}

【业务描述】
{{description}}

【目标】
生成一张前端到后端到数据层的完整流程图，突出用户触发、前端处理、接口层、服务层、数据层、异常分支。

【强制结构要求】
1. 必须按 4 层或以上进行分组：
   - Frontend
   - Gateway / BFF / API Layer
   - Backend Services
   - Database / Cache / External Systems
2. 必须包含：
   - 用户触发
   - 前端校验或状态处理
   - 接口请求
   - 服务处理
   - 数据读写
   - 返回结果
   - 至少一个异常处理分支
3. 必须展示至少一个跨层调用链。
4. 必须让整张图适合大画布展示，层次清晰，可继续扩展。

【增强要求】
- 自动补齐合理的鉴权、日志、缓存、失败兜底节点
- 连线要体现明确关系，不要全部无语义
- 节点标题简短专业，避免长句
- 结构优先专业系统图，而不是教学示意图

【输出倾向】
- 主链路清晰
- 分层明显
- 可编辑性强
- 适合 React Flow 画布继续加工
