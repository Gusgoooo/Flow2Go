# Mind Map Template（思维导图）

## 适用场景
适用于：主题分支展开、层级联想、知识梳理、概念拆解、把一段自然语言或文档整理成树状结构。

## 模板目标
生成一张思维导图：
- 只有节点，所有内容都通过节点层级表达
- 层级通过父节点 -> 子节点表达
- 不允许任何画框/子图：Mermaid 输出中不得包含 `subgraph` / `end`
- 输出结果应优先体现“中心主题 + 一级分支 + 二级分支”的脑图结构，而不是流程链或句子堆叠

## 强约束
1. Mermaid 输出中 **禁止出现任何** `subgraph` 或 `end`。
2. 只能包含普通节点 `id[中文标题]` 与边 `A --> B`，**默认不要使用动作边**。
3. 必须使用 `flowchart LR`。
4. 必须形成单根树状结构：全图只能有一个中心根节点。
5. 所有一级分支都必须直接从根节点发散，不能把同级主题串成链。
6. 所有二级分支都必须挂载在对应一级分支下，不能跨分支连接。
7. 禁止回边、环、交叉引用、跨层乱连。
8. 节点 label 必须为中文且尽量短，优先使用名词、短语、主题词，不要使用完整句子。
9. 语义太长时用 V2：`主标题｜副标题`，但仍应尽量短，不要把长解释写进节点。
10. 如果输入内容很多，优先压缩和归类，不要把每个细节都展开成节点。

## 生成前的结构化规则（高优先级）
在输出 Mermaid 之前，先在内部完成以下步骤：

1. 先提炼 **唯一中心主题**
   - 全图只能有一个根节点
   - 根节点必须是文章、段落或主题的最高层概括

2. 再抽取 **一级主题**
   - 一级主题应是 3~6 个大的分类、维度或模块
   - 一级主题必须彼此并列
   - 一级主题不能写成步骤顺序

3. 再抽取 **二级主题**
   - 二级主题应是对一级主题的进一步拆解
   - 只保留关键子点，不要把所有原文细节都展开
   - 同一一级主题下的二级节点建议不超过 4 个

4. 优先归类，禁止流水账
   - 不要按原文语序逐句生成节点
   - 不要把段落句子直接改写成节点
   - 要先把语义归类成“主题 -> 子主题 -> 要点”

5. 优先树状发散，禁止链式冒充脑图
   - 错误：`根 -> A -> B -> C`
   - 正确：`根 -> A`，`根 -> B`，`根 -> C`

## 内容约束
1. 节点应优先是：
   - 主题词
   - 分类名
   - 维度名
   - 要点短语
2. 节点应避免是：
   - 完整句子
   - 动作描述
   - 因果链条
   - 流程步骤
3. 如果原文中有多个近义项或相似项，应先合并后再表达。
4. 如果某一层同级内容过多，应优先归并为一个上位概念，而不是全部平铺。

## 层级约束
1. 推荐深度为 2~4 层。
2. **不要为了满足层级而强行造孙节点。**
3. 只有当某个子主题确实还能自然拆分时，才生成孙节点。
4. 如果输入本身更适合 2 层或 3 层表达，则不要硬扩展到更多层。

## 数量约束
- 节点总数建议 8~18 个
- 一级分支建议 3~6 个
- 每个一级分支下建议 1~4 个二级节点
- 超过上限时，优先删减叶子节点，而不是继续增加层级

## 连线约束
1. 默认只使用 `A --> B`
2. 除非用户明确要求，不要使用 `A -->|动作| B`
3. 连线只表示父子归属，不表示动作、调用、流转、审批、回流
4. 不允许跨一级分支互连
5. 不允许多个同级节点串成一条长链

## 布局与样式说明
布局与样式由系统 `mind-map` 布局器负责：
- 列式排版：根在最左列，深度越深越往右；同列纵向排列
- 列间距目标约 8 个 grid units
- 节点描边按“深度层级”顺序取主题色，描边粗细 `strokeWidth=2`
- 边使用贝塞尔曲线效果

## 输出倾向
- 优先输出“中心主题 + 并列一级分支 + 少量二级分支”的稳定脑图
- 优先清晰分类，不优先覆盖所有细节
- 优先短标签，不优先长解释
- 优先自然树状结构，不优先追求层数
