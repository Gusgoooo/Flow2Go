{{topic}}

【流程说明】
{{description}}

【输出目标】
生成一张完整审批流，体现发起、校验、逐级审批、条件判断、退回补充、通知归档等关键环节。

【强制结构要求】
1. 必须按角色或阶段进行分组，建议包含：
   - Applicant / Requester
   - Manager / Reviewer / Approver
   - System / Notification
   - Archive / Record
2. 必须包含：
   - 提交申请
   - 至少 2 级审批
   - 至少 1 个判断节点
   - 至少 1 条退回 / 补充分支
   - 最终通知或归档节点
3. 必须体现审批状态流转，不要只画操作动作。
4. 连线要突出提交、审批、驳回、退回、通知、归档等语义。

【增强要求】
- 自动补齐材料校验、超时提醒、会签、日志、归档等合理节点
- 结构应清晰、专业，适合管理流程展示
- 大画布上要具备扩展空间
- 避免过度简化成直线流程

【输出倾向】
- 企业流程图风格
- 状态清晰
- 判断明确
- 适合继续编辑
